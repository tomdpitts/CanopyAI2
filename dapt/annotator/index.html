Crown Box Annotator

Correctness rules (pipeline-critical)

  • Exhaustive per tile — box every crown, or mark the tile empty. Partially annotated tiles corrupt the metric; the per-tile status column tracks this.
  • Box each crown separately, even when crowns touch or overlap — overlapping boxes are expected and valid.
  • Boxes tight to the crown's visible extent. Include edge-clipped crowns: box the visible part.
  • Dryland crowns can be small (~20 px) — zoom in for tight placement.

Annotating

  • Click-drag on the image to draw a box (label is always Tree; no dialog — you're immediately ready to draw the next).
  • Click a box to select it; drag it to move, drag its handles to resize.
  • Shift+drag always draws a new box, even starting inside an existing one (needed for overlapping crowns).
  • A tile is "done" when it has ≥1 box or is explicitly marked empty. "Unannotated" is a distinct third state.
  • Marking a tile empty requires zero boxes; drawing a box clears the empty mark.

View

  • Mouse wheel = zoom at cursor. Rendering is nearest-neighbour at native resolution — coordinates are always exact image pixels, at any zoom.
  • Pan: drag with middle or right mouse button, or hold Space and drag.

Data & CSV

  • Work autosaves to this browser (localStorage), keyed by filename — refresh/crash safe. Reload the same folder to resume.
  • Export CSV writes image_path,xmin,ymin,xmax,ymax,label,domain: one row per box (integer pixels, origin top-left), and exactly one all-blank-fields row per empty-marked tile. Unannotated tiles are excluded (with a warning).
  • Import CSV restores boxes, empty marks and per-tile domains by filename (replaces that file's current annotations) — use it to resume on another machine or after clearing the browser. Imported per-tile domains are preserved on export; the "default domain" field only applies to tiles without one.
